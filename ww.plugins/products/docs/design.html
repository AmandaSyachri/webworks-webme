Templates

Product templates define how the product will look on the front-end of the site.

If you don't define a template, then all the product fields will be shown in a tabular format.

There are two templates for each type - a "multi-view" template, and a "single-view" template. The multi-view template is shown on pages where more than one product may be visible, and the single-view template is used on pages dedicated to displaying a single product.

A product template is designed much the same as a web-page template.

Let's say you have the following fields in your product type: email, name, address, phone

You can display this as a table using code like this:

<table
	<tr><th>Name</th><td>{{$name|escape}}</td></tr>
	<tr><th>Email</th><td><a href="mailto:{{$email|escape}}">{{$email|escape}}</a></td></tr>
	<tr><th>Phone</th><td>{{$phone|escape}}</td></tr>
	<tr><th>Address</th><td>{{$address}}</td></tr>
</table>

The variables names are equal, including case, to the product field names.

Notice the |escape bits. If someone enters something like great <3 stuff in the name, email or phone fields, then WebME should convert the < to &lt; before pasting to the HTML. This is called "escaping".

We don't use |escape on the address or any other textarea fiel, as these are usually filled in with rich-text editors so are fine to output directly.
